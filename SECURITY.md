## 🐛 报告漏洞

我们非常重视安全问题。如果你发现了安全漏洞，请**不要**公开披露。

### 报告流程

1. **私密报告**: 发送邮件到 security@example.com
2. **包含信息**:
   - 漏洞类型
   - 受影响的版本
   - 重现步骤
   - 潜在影响
   - 建议的修复方案（如果有）

3. **响应时间**:
   - 我们会在 48 小时内确认收到报告
   - 在 7 天内提供初步评估
   - 根据严重程度，在 30-90 天内发布修复

### 安全更新

- 安全补丁会优先处理
- 修复后会发布新版本
- 在 CHANGELOG 中标注安全修复
- 严重漏洞会发布安全公告

## 🛡️ 安全最佳实践

### 用户建议

1. **保护配置文件**
   - `ftp-configs.json` 包含 FTP 密码
   - 不要分享或提交到版本控制
   - 定期更改 FTP 密码

2. **使用安全连接**
   - 优先使用 FTPS（启用"使用安全连接"选项）
   - 避免在公共网络使用

3. **及时更新**
   - 保持应用程序最新版本
   - 关注安全公告

4. **权限控制**
   - 使用最小权限的 FTP 账户
   - 限制 FTP 账户的目录访问

### 开发者建议

1. **依赖管理**
   - 定期更新依赖包
   - 使用 `npm audit` 检查漏洞
   - 及时修复已知漏洞

2. **代码审查**
   - 所有 PR 需要代码审查
   - 关注安全相关的变更
   - 避免硬编码敏感信息

3. **安全测试**
   - 测试输入验证
   - 检查路径遍历漏洞
   - 验证文件权限

## 🔐 已知限制

1. **密码存储**: 配置文件中的密码以明文存储
   - **风险**: 如果系统被入侵，密码可能泄露
   - **缓解**: 使用操作系统的文件权限保护配置文件
   - **计划**: 未来版本将使用加密存储

2. **网络传输**: 标准 FTP 不加密
   - **风险**: 密码和数据可能被窃听
   - **缓解**: 使用 FTPS 选项
   - **建议**: 始终启用安全连接
